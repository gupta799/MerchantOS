MerchantOS Architecture

Computer-use simulation lab for agentic commerce readiness

MerchantOS runs autonomous computer-use agents against a storefront, captures deep telemetry, and turns the trace into merchant-owned optimization guidance for agentic commerce, MCP tools, and computer-use reliability.

What The Harness Is

  • It owns the simulation objective, safety policy, telemetry schema, and evaluation loop.
  • DeepAgents/Gemma can be the harness brain, but the harness is bigger than one model.
  • It decides what to send to the CUA provider and how to interpret the result.

Where Kernel Fits

  • Kernel is the managed browser execution environment.
  • Local mode uses the React SDK websocket; Kernel mode uses cloud browser controls.
  • The same trace, policy, and telemetry services work in both modes.

Why Merchants Care

  • They learn how AI buying agents experience their storefront.
  • They see concrete failures: loops, no-op clicks, hidden state, missing structured data.
  • They get an MCP/tool roadmap that keeps commerce merchant-owned.
